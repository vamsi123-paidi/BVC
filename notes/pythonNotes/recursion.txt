🔁 Recursion (Complete Deep Understanding)

---

📖 1. What is Recursion?

Recursion is when a function calls itself to solve a problem.

👉 Simple meaning:

> Break a problem into smaller parts and solve it using the same logic

---

🧠 2. Core Idea (MOST IMPORTANT)

Every recursive function MUST have:

✅ 1. Base Case

👉 Condition to STOP recursion

✅ 2. Recursive Case

👉 Function calling itself

---

⚠️ Without base case → Infinite loop (crash)

---

📌 3. Basic Example (Understanding Flow)

python
def count(n):
    if n == 0:   base case
        return
    print(n)
    count(n - 1)   recursive call


Call:

python
count(3)


---

🔍 Execution Flow


count(3)
 → print 3
 → count(2)
   → print 2
   → count(1)
     → print 1
     → count(0) → stops


---

📤 Output


3
2
1


---

🧠 4. How Recursion Works Internally

👉 Uses call stack

Each function call is stored in memory:


count(3)
count(2)
count(1)
count(0)


Then it starts returning back

---

🔥 5. Example 1: Factorial (VERY IMPORTANT)

📖 Concept

Factorial:


5! = 5 × 4 × 3 × 2 × 1


---

Code

python
def fact(n):
    if n == 1:   base case
        return 1
    return n * fact(n - 1)


---

🔍 Flow (fact(4))


fact(4)
= 4 * fact(3)
= 4 * (3 * fact(2))
= 4 * (3 * (2 * fact(1)))
= 4 * (3 * (2 * 1))
= 24


---

📤 Output


24


---

🧠 Insight

👉 Recursion builds result while returning

---

🔢 6. Example 2: Sum of Numbers

python
def sum_n(n):
    if n == 0:
        return 0
    return n + sum_n(n - 1)


---

📤 Output

python
sum_n(5) → 15


---

🔁 7. Example 3: Reverse a String

python
def reverse(s):
    if s == "":
        return ""
    return s[-1] + reverse(s[:-1])


---

📤 Output


reverse("abc") → "cba"


---

🔍 8. Example 4: Fibonacci (Important)

python
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)


---

📤 Output


fib(5) → 5


---

⚠️ Important

👉 Fibonacci recursion is slow (repeats work)

---

🧠 9. When to Use Recursion

Use when:

* Problem can be divided into smaller same problems
* Tree/graph problems
* Backtracking

---

🌍 Real-World Examples

* Folder traversal
* File search
* Menu systems
* Game logic

---

⚖️ 10. Recursion vs Loop

| Recursion                 | Loop                 |
| ------------------------- | -------------------- |
| Function calls itself     | Uses iteration       |
| Uses stack                | Uses variables       |
| Cleaner for some problems | Faster in most cases |

---

⚠️ 11. Common Mistakes

❌ No base case
❌ Wrong recursive call
❌ Infinite recursion
❌ Stack overflow

🎯 13. Step-by-Step Thinking Method

When solving recursion:

1. Identify base case
2. Break problem
3. Assume smaller problem works
4. Build final result

---

💥 Golden Rule

👉 Think like this:

> “If I already know solution for smaller input… how do I build current answer?”
