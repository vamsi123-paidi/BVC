🧠 1. What is OOP?

📖 Definition

Object-Oriented Programming (OOP) is a way of writing programs using **objects and classes** to organize code.

---

🧠 Why OOP Exists (Important Insight)

Problem in normal coding:

* Code becomes messy
* Repetition increases
* Hard to manage large applications

👉 OOP solves this by:

* Structuring code
* Reusing logic
* Modeling real-world entities

---

🌍 Real-Life Analogy

| Real World              | Programming |
| ----------------------- | ----------- |
| Car                     | Object      |
| Blueprint               | Class       |
| Features (color, speed) | Attributes  |
| Actions (start, stop)   | Methods     |

---

   * Attributes → ?
   * Methods → ?

🏗️ 2. Class & Object

---

📖 Class

Blueprint/template for creating objects

📖 Object

Instance of a class

---

🧠 Simple Understanding

👉 Class = design
👉 Object = real item

---

📌 Example

python
class Student:
    name = "Vamsi"

print(s1.name)


---

🧠 Key Insight

* Class defines structure
* Object accesses data

⚙️ 3. Constructor (`__init__`)

---

📖 Definition

A special function that runs automatically when object is created

---

📌 Example

python
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

s1 = Student("Vamsi", 22)
print(s1.name)


---

🧠 Important Concepts

* `self` → current object
* Used to store values

---

🧠 Real-Life Analogy

👉 When a student joins college → details are assigned automatically

---

🧪 Practice Tasks

1. Create class Employee (name, salary)
2. Print details

🧱 5. Instance vs Class Variables

---

📖 Instance Variable

Specific to each object

📖 Class Variable

Shared across all objects

---

Example

python
class Student:
    school = "ABC"   class variable

    def __init__(self, name):
        self.name = name   instance variable


---

🧠 Insight

👉 Instance → unique
👉 Class → common

---

🔒 6. Encapsulation (Basic)

---

📖 Definition

Hiding internal data and restricting access

---

📌 Example

python

        self.__x = 10

---

🧠 Concept

* `__x` → private variable
* Cannot access directly

---

🧠 Why Important

* Protect data
* Avoid misuse

---

🧬 1. Inheritance

---

📖 Definition

One class can use properties of another class

---

🧠 Why Needed

* Code reuse
* Avoid duplication

---

📌 Example

python
class Parent:
    def show(self):
        print("Parent")

class Child(Parent):
    pass

c = Child()
c.show()


---

🧠 Real-Life

👉 Child inherits from parent

---

🧪 Practice Tasks

1. Create Animal → Dog class
2. Use inherited method

---

🔄 2. Method Overriding

---

📖 Definition

Child class modifies parent method

---

Example

python
class Parent:
    def show(self):
        print("Parent")

class Child(Parent):
    def show(self):
        print("Child")


---

🧠 Insight

👉 Same method → different behavior

---

🧪 Practice Tasks

1. Override method
2. Call child method

🔁 3. Polymorphism

---

📖 Definition

Same function behaves differently

---

Example

python
def add(a, b):
    return a + b

print(add(2, 3))
print(add("a", "b"))


---

🧠 Insight

👉 Same function → multiple forms

---

🧪 Practice Tasks

2. Observe behavior

🧩 4. Abstraction

---

📖 Definition

Hiding implementation details and showing only essential features

---

Example

python
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass

🔍 6. super() Function

---

📖 Definition

Used to access parent class methods

---

Example

python
class Parent:
    def __init__(self):
        print("Parent")

class Child(Parent):
    def __init__(self):
        super().__init__()
        print("Child")


---

🧠 Insight

👉 Avoid rewriting parent code

---

🧪 Practice Tasks

1. Use super() in constructor
2. Observe output order
